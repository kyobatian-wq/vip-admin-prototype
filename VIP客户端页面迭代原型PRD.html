一、需求背景与目标

1.1 背景

当前 DD.GG 客户端 VIP 页面仅包含"Bônus""Bônus Diário""Bônus Semanal""Bônus Mensal"四个页签，缺少 Panda 客户端已有的"Instruções para manter o nível"（保级说明）页签，用户无法在客户端直接查看保级规则、保级等级表和完整的VIP规则说明。此外，现有奖励领取按钮缺乏清晰的状态区分（是否达标可领取），也没有对应的引导/成功反馈弹窗。

1.2 目标

  • 补充"Instruções para manter o nível"页签，对齐 Panda 客户端的保级说明呈现
  • 完善奖励领取按钮的状态设计：区分"未达标禁用"与"已达标可领取"，并细化已达标后的二次校验逻辑
  • 补充两个关键弹窗：充值引导弹窗、领取成功弹窗
  • 明确规则文案中哪些内容为静态模板、哪些内容需与后台配置动态绑定（稽核倍数、生效场馆等）

三、页面结构总览

页面路径：客户端 App ＞ 我的 ＞ VIP。页面自上而下分为：① 顶部等级卡片（当前等级、下一等级所需流水、Coletar Tudo/Histórias入口）② Tab导航栏（Bônus / Bônus Diário / Bônus Semanal / Bônus Mensal / Instruções para manter o nível）③ 对应Tab的内容区。

  • Bônus：晋级奖励列表，含当前在途等级的进度条
  • Bônus Diário / Semanal / Mensal：对应周期的等级礼金列表 + 领取按钮
  • Instruções para manter o nível：本次新增，含保级规则说明、保级等级表、完整VIP规则说明
Tab栏支持横向滚动；当某个Tab下存在"已达标可领取"的奖励时，Tab文字右侧显示红色提示点，未领取状态解除后红点自动消失。

四、功能详细设计

4.1 顶部等级卡片

功能说明：展示用户当前VIP等级、距下一等级所需流水，并提供快捷入口。

字段/元素说明
Nível Atual 标签红色角标，标注当前等级卡片状态
等级图标 + 等级号圆形皇冠图标，图标颜色随等级区间变化（如V0-V2银色、V3+金铜色）
Siga VIPx提示用户当前正在向哪个等级迈进
Ainda precisa apostar距下一等级所需的剩余流水金额，读取「晋级需流水－当前累计流水」
Coletar Tudo一键领取入口，点击后走与其他领取按钮相同的二次校验逻辑（见4.6）
Histórias跳转历史领取记录页（本期不展开设计）

4.2 Tab导航与红点提示

  • Tab顺序：Bônus / Bônus Diário / Bônus Semanal / Bônus Mensal / Instruções para manter o nível
  • 当前Tab有可领取（已达标）奖励时，Tab文字右侧显示红色圆点；无可领取奖励时不显示
  • 「Instruções para manter o nível」为纯说明性页签，不产生红点提示

4.3 Bônus（晋级奖励）

功能说明：展示各等级的晋级所需流水与晋级奖励金额，并对当前"在途"等级展示实时进度。

  • 列表字段：等级、晋级所需流水（或进度条）、晋级奖励金额
  • 当前用户正在冲刺的等级，所需流水列展示为进度条，下方附文字"已完成金额 / 所需金额"
  • 底部「Reclamar Recompensa de Upgrade」按钮，状态逻辑见4.6

4.4 Bônus Diário / Semanal / Mensal

功能说明：分别展示日/周/月礼金在各等级下的金额，供用户对照当前等级查看可得奖励。

  • 列表字段：等级、对应周期奖励金额
  • 底部分别为「Reivindicar Recompensa Diária / Semanal / Mensal」按钮，状态逻辑见4.6

4.5 Instruções para manter o nível（保级说明，本次新增）

功能说明：补充客户端保级相关的规则说明与等级表，对齐 Panda 客户端呈现。

4.5.1 保级规则文案

固定展示以下三段说明（读取后台「规则说明→保级说明」的默认文案或自定义富文本）：

  • 1. 保级判定时点与规则：每月1号00:00判定，上月充值/流水达标则保级成功，否则降1级
  • 2. 降级后重新达标的处理：需重新累计充值/流水才能再次晋级，且晋级奖励不重复发放
  • 3. 当月内的补救说明：当月内重新达到原等级要求可恢复，但同样不重复发放晋级奖励
该三段文案的具体措辞以后台「规则说明→保级说明」实际配置内容为准（默认或运营自定义的富文本），此处仅描述其功能定位。

4.5.2 保级等级表（Tabela de níveis de manutenção）

展示各VIP等级对应的保级门槛，字段：等级、保级需充值、保级需流水。

数据来源：后台「VIP等级配置」中每个等级的「保级需充值」「保级需流水」字段，逐级取值；表格行数与后台实际配置的等级数量一致（新增/删除等级会同步影响此表行数）。

4.5.3 Instruções sobre regras VIP（完整规则说明）

在保级等级表下方，展示完整的VIP规则说明文案，包含晋级说明、日/周/月奖励说明、奖励过期处理、稽核说明、活动声明、解释权声明。其中两处内容需与后台配置动态绑定：

文案片段动态数据来源
"rollover Nx"（稽核倍数）读取后台「等级条件与流水稽核」中配置的「稽核倍数」字段，N随后台配置实时变化
"Slot：[平台清单]"（生效场馆）读取客户端「生效场馆」配置，与后台「稽核限定平台」勾选结果联动；若后台选择"不限制"，此处应展示"不限制游戏平台"或隐藏该句
活动声明、解释权声明两段为固定合规文案，不随任何后台配置变化。

4.6 领取按钮状态与弹窗逻辑

按钮状态说明：

状态视觉表现可点击性点击后行为
未达标灰色/暗橄榄色，文字降低对比度不可点击（禁用）无任何响应，不弹窗
已达标 - 区间内未充值高亮黄色，与已充值状态视觉一致可点击弹出「充值引导弹窗」，提示完成任意充值后可领取
已达标 - 区间内已充值高亮黄色可点击弹出「领取成功弹窗」，展示到账金额
关键说明："已达标"仅代表流水/等级等基础条件已满足，按钮因此保持高亮可点击；但点击后是否真正发放奖励，还需二次校验"统计区间内是否有充值行为"这一独立条件。两种已达标状态在按钮外观上没有区别，用户需点击后才能得知具体结果。

弹窗一：充值引导弹窗

  • 触发条件：按钮为"已达标"状态，但点击时校验到区间内未充值
  • 内容：插画 + 文案"Receba uma recompensa exclusiva após efetuar qualquer depósito."
  • 按钮：「Depósito」（主按钮，跳转充值页）、「Cancelar」（次按钮，关闭弹窗）

弹窗二：领取成功弹窗

  • 触发条件：按钮为"已达标"状态，且点击时校验到区间内已充值
  • 内容：成功图标（绿色对勾）+ 文案"Recompensa recebida com sucesso!" + 到账金额
  • 按钮：「Confirmar」，点击关闭弹窗

五、关键交互流程

5.1 点击奖励领取按钮的完整判断流程

  • ① 用户点击某个奖励领取按钮
  • ② 若按钮为"未达标"禁用状态 → 无任何响应，流程结束
  • ③ 若按钮为"已达标"状态 → 系统校验"当前统计区间内是否有充值行为"
  • ④ 校验结果为"未充值" → 弹出充值引导弹窗，用户可选择去充值或取消
  • ⑤ 校验结果为"已充值" → 直接发放对应奖励，弹出领取成功弹窗，展示到账金额

5.2 Instruções para manter o nível 页签加载流程

  • ① 用户切换到「Instruções para manter o nível」页签
  • ② 页面加载后台「规则说明→保级说明」的当前文案（默认或自定义富文本）
  • ③ 加载「VIP等级配置」中所有等级的保级需充值/需流水，渲染保级等级表
  • ④ 加载完整VIP规则说明文案，其中"稽核倍数""生效场馆"两处按当前后台配置动态填充后展示

七、异常与边界情况

场景处理方式
用户在充值引导弹窗中点击「Depósito」后未完成充值返回VIP页面时按钮仍为"已达标未充值"状态，可再次点击触发同样的引导弹窗
后台「稽核限定平台」设置为"不限制"规则文案中的Slot清单需替换为"不限制游戏平台"或整句隐藏，避免展示空列表
后台等级数量发生变更（新增/删除）客户端保级等级表需实时同步行数，不需要客户端单独维护等级数据
用户在同一天内连续跨越多个等级晋级奖励按4.3/5.1规则持续累加，点击领取时一次性合并发放
规则说明富文本内容包含链接/列表等复杂排版客户端需支持富文本渲染，而非纯文本展示，避免格式丢失

八、验收标准

  • ☐ 客户端VIP页面新增「Instruções para manter o nível」页签，且位于Tab导航最后一位
  • ☐ 保级等级表行数与后台「VIP等级配置」的等级数量一致，数值正确对应"保级需充值/需流水"
  • ☐ 规则说明文案中的"rollover Nx"与后台「稽核倍数」配置值一致，修改后台配置后客户端展示同步更新
  • ☐ 规则说明文案中的"Slot：[平台清单]"与后台「稽核限定平台」勾选结果一致
  • ☐ 未达标的领取按钮为禁用态，点击无任何响应，不弹出任何弹窗
  • ☐ 已达标且区间内未充值时，点击按钮弹出充值引导弹窗，而非直接发放奖励
  • ☐ 已达标且区间内已充值时，点击按钮弹出领取成功弹窗，并展示正确的到账金额
  • ☐ 有可领取奖励的Tab正确显示红点提示，领取/条件变化后红点能正确消失
  • ☐ Bônus页签中在途等级的进度条数值与实际累计流水/所需流水一致
