一、背景与目标

返水（Rebate）是面向玩家的投注返现激励功能：玩家在各游戏场馆（Slot、Cassino 等）产生投注后，系统按场馆设定的阶梯比例计算可返还金额，玩家可在返水页面查看当日/次日可领取额度，按场馆查看具体返水比例规则，并将已产生的返水一键领取到主账户余额。本次迭代在原有 Slot 场馆返水列表基础上，补充了 Cassino 游戏类型入口、场馆卡片的返水进度可视化、领取结果确认弹窗、以及历史记录查询页，目标是让玩家更清晰地感知"投入-进度-可领取-已领取"这条链路，提升返水功能的可用性与领取转化率。

二、功能范围

范围内容
本次新增 / 调整 Cassino 游戏类型入口；场馆卡片返水进度条（底纹样式）；点击 Receber 后的领取确认弹窗；Histórico 历史记录页（含时间范围筛选）；顶部 Tab 栏中 Rebate / Histórico 的改动标识；额度汇总条新增「返水领取模式配置」（次日领取 / 实时领取二选一，见 5.1 节）。
沿用原有设计 顶部一级 Tab 栏（Eventos / Rebate / VIP / Código de Resgate / Histórico / Pendente）；场馆返水详情页（Taxa de reembolso）及其筛选下拉；底部主导航。
不在本次范围 Eventos、VIP、Código de Resgate、Pendente 等其他一级 Tab 页面的功能设计；具体返水比例的计算与结算后端逻辑；Esportes（体育）返水规则的详细定义。

五、页面一：返水首页

5.1 页面结构

页面自上而下分为四个区域：顶部一级 Tab 栏、返水额度汇总条、主体区（左侧游戏类型 / 操作栏 + 右侧场馆列表）、底部主导航。

区块元素说明
顶部 Tab 栏 一级 Tab Eventos / Rebate / VIP / Código de Resgate / Histórico / Pendente，横向可滚动；当前页面 Rebate 为选中态（黄色下划线 + 加粗）。
Tab 标识 Rebate、VIP 携带红色小圆点提示（新消息 / 待处理提示）；本次迭代改动范围：Rebate、Histórico 两个 Tab 叠加橙色底纹高亮 + 红色「Ajustado」角标，用于区分本轮调整涉及的入口，验收后可移除该标识。
本次新增 额度汇总条 两组数值 + 说明文案 数值为玩家对所有场馆返水的汇总（非单一场馆数值）。本次新增：该区域按后台配置的「返水领取模式」二选一渲染，玩家侧无法自行切换，两种模式互斥：
配置模式字段一字段二内部说明文案（不展示给玩家）Receber 逻辑
次日领取 Pode ser coletado hoje（今日可领取） Disponível para coleta amanhã（次日预计可领取） * Rebate é liquidado em D+1: os valores de hoje referem-se às apostas de ontem. 当日额度为固定批次金额，领取后归零，新额度需等次日结算才会出现。

5.2 场馆卡片字段说明

字段说明
场馆图标 / 名称如 PG、TaTa、JDB、Spribe、PP、CQ9、BAO、FC 等（Slot 类型示例场馆，实际以接口返回为准）。
Aposta玩家在该场馆的累计投注额，按自然日实时计算，次日 00:00 重置清零重新累计。
Taxa de reembolso玩家当前已产生的综合返水比例（示例默认 0,00%，随投注增加而变化）。
本次新增 返水进度条 胶囊形进度条，底纹为深灰色（#3a3a3a），文案「Aposte 1,00 para receber X%」直接叠加显示在进度条底纹之上；进度条填充部分（橙黄色渐变）随玩家投注推进逐步填充。
计算公式（已确认）：距下一档 —— 填充比例 = 当前投注额 ÷ 下一档所需投注额（阶梯门槛沿用第六节场馆详情页的投注门槛表：1,00 / 100.000,00 / 500.000,00 / 1.000.000,00 / 3.000.000,00 / 5.000.000,00 / 10.000.000,00），达到或超过最高档时封顶显示 100%。当 Aposta 为 0 时填充为 0%，仅显示底纹。原型中 PG（45.000,00 / 45%）、JDB（320.000,00 / 64%）两张卡片已按该公式演示非零进度状态，其余卡片保持 0% 占位。
Coletável该场馆当前可领取的返水金额，原型中按「Aposta × Taxa de reembolso」估算展示，实际以接口返回为准。
Chevron（›）点击整张卡片或箭头，跳转至该场馆的返水详情页（Taxa de reembolso）。
场馆列表排序规则、Aposta 的统计周期均已在第十一节确认（详见 11.1）；各场馆真实的阶梯门槛与比例数值由后端接口写入，本文档及原型中的数值仅为演示占位。

六、页面二：场馆返水详情页（Taxa de reembolso）

由返水首页点击任意场馆卡片进入，用于查看该场馆完整的投注阶梯与对应返水比例规则。

元素说明
返回按钮左上角「‹」，点击返回返水首页。
标题固定文案「Taxa de reembolso」。
筛选下拉一：Modalidade 可选项 Slot / Esportes；默认取值为进入前所选的游戏类型（Slot）。切换为 Esportes 时，当前无阶梯数据，展示统一缺省页（见下方"空状态"说明）。
空状态（已确认） Esportes 暂无阶梯数据时，展示统一的缺省页：灰度文档 + 放大镜图标，下方文案「Sem Registros」，居中显示；与场馆列表、历史记录的空状态样式保持一致。
筛选下拉二：Fornecedor 展示为场馆宫格选择器（示例 17 家场馆：PG、Tada、JDB、Spribe、PP、CP、G759、FaChai、POPOK、PLAYSON、RUBYPLAY、inout、FASTSPIN、PANDA、TOPPLAYER、PLAYNGO、Betby），当前选中项高亮；切换后下方表格数据联动刷新。
阶梯表格 列：Modalidade / Fornecedor / Aposta（投注门槛）/ Taxa de reembolso（对应比例）。示例阶梯（数值为占位）：
10.000.000,00 → 5,00%；5.000.000,00 → 3,00%；3.000.000,00 → 2,00%；1.000.000,00 → 1,20%；500.000,00 → 0,80%；100.000,00 → 0,50%；1,00 → 0,30%。
已确认：各场馆真实的阶梯门槛与比例数值由后端接口写入，前端仅负责按接口返回渲染，不做本地硬编码。

七、交互组件：领取（Receber）确认弹窗

本次新增 玩家在返水首页点击绿色 Receber 按钮后触发，用于二次确认领取动作并展示领取结果。

元素说明
触发方式点击返水首页左侧「Receber」按钮。
顶部插画区营销风格插画（示意为吉祥物 + 金币元素），用于强化领取成功的正反馈氛围，具体素材由视觉设计提供。
提示文案「Parabéns por ganhar R$ X,XX em fichas de bônus」，金额取自当前「Pode ser coletado hoje」的汇总值。
结算周期（已确认，修正） 返水为当日实时计算、手动领取，并非隔日（D+1）结算（此前版本描述有误，已修正）。玩家的返水额度随投注实时累计；打开 Receber 领取入口时，按当前快照金额展示 —— 即"进入领取页有多少显示多少，领取多少"：弹窗打开瞬间读取一次当前可领取总额并锁定展示，点击 Confirmar 即按该快照金额领取。若打开弹窗后、点击 Confirmar 前又产生了新的返水，新增部分不计入本次领取，会保留在「Pode ser coletado hoje」中供下次领取，不会丢失，以此方式规避实时计算下的并发冲突。
Confirmar 按钮点击后：① 关闭弹窗；② 将该笔（快照）金额计入玩家主账户余额（需后端接口支持）；③ 返水首页「Pode ser coletado hoje」按领取金额扣减（若领取期间无新增，则归零）；④ 对应场馆卡片的 Coletável 应同步扣减（原型中暂未做单卡片联动，需前端实现时补充）。
无可领取金额时（已确认）若「Pode ser coletado hoje」为 0,00，Receber 按钮置灰禁用、不可点击（原型已实现：领取成功后按钮自动变灰）。

八、页面三：历史记录页（Histórico）

本次新增 用于查询玩家历史返水领取流水，入口为返水首页左侧「Histórico」按钮或顶部「Histórico」一级 Tab。

元素说明
顶部 Tab 栏与返水首页一致，当前 Histórico 为选中态；点击「Rebate」Tab 可返回返水首页。
时间范围筛选 左上角下拉按钮，默认「Hoje」，点击展开二列宫格选项：Hoje / Ontem / Últimos 7 Dias / Últimos 30 Dias。选中后按钮文案更新为所选项。
已确认：切换时间范围时重新请求接口获取对应周期的数据，而非前端本地过滤已加载的数据。原型中已用短暂的「Carregando…」加载态模拟这一请求过程（实际接口对接时替换为真实请求）。
Bônus 汇总右上角展示玩家在当前筛选周期内累计已领取的返水奖金总额。
流水表格 列：Tempo（领取时间）/ Nome do Evento（事件名称，如 Cashback de Apostas）/ Evento de Recompensa（本笔到账金额）/ Fonte（来源，固定为 Rebate）。
空状态（已确认）所选时间范围内无记录时（原型以「Ontem」为演示），展示统一缺省页：灰度文档 + 放大镜图标，下方文案「Sem Registros」，居中显示；与返水首页场馆列表、场馆详情页 Esportes 空状态样式保持一致。

十一、待确认事项

11.1 已确认结论

问题结论
返水进度条的填充比例计算公式距下一档：填充比例 = 当前投注额 ÷ 下一档所需投注额，封顶 100%。
本次新增 返水领取模式（结算周期） 系统支持两种后台可配置的领取模式，二选一生效，玩家侧不感知切换：
① 次日领取：额度为固定批次金额，按 D+1 结算，领取后归零、需等次日才有新额度（对应字段 Pode ser coletado hoje / Disponível para coleta amanhã）；
② 实时领取：当日实时计算、手动领取，对应字段 Total gerado hoje（今日累计产生）/ Já coletado hoje（今日已领取），可领取余额 = 产生总额 − 已领取。打开 Receber 弹窗时按当前余额快照展示并锁定，即"进入领取页有多少显示多少，领取多少"；弹窗打开期间新产生的返水不计入本次、保留待下次领取，以此规避实时计算下的并发冲突。
可领取金额为 0 时 Receber 按钮的表现置灰禁用，不可点击（而非点击后弹窗提示）。
两种领取模式的说明文案是否展示给玩家不展示。D+1 与实时计算的说明文案仅作为内部逻辑备注，客户端界面不呈现给玩家，前端已隐藏对应元素。
场馆列表排序规则由后台接口控制返回顺序，前端直接按接口顺序渲染，不做二次排序。
各场馆返水阶梯真实数值由后端接口写入，前端按接口返回渲染，本文档 / 原型中的数值均为演示占位。
历史记录时间筛选的数据加载方式重新请求接口，非前端本地过滤。
三类空状态（Cassino 场馆为空 / Esportes 阶梯为空 / 历史记录周期无数据）统一使用同一套缺省页：灰度文档 + 放大镜图标，文案「Sem Registros」，居中显示。
Tab 改动标识（Ajustado 角标）仅用于本轮内部评审阶段标识改动范围，不作为正式功能保留，上线前需移除。
Aposta 的统计周期口径 Aposta（投注额）按自然日实时计算，次日 00:00 重置计数清零重新累计；但已计算出、尚未被玩家领取的返水奖金（Coletável）会持续累计保留，不随 Aposta 清零而消失或重置，直到玩家手动领取为止。即"Aposta 按天清零，Coletável 跨天累计"。
本轮全部待确认事项已确认完毕，11.1 表为最终结论，供开发排期与联调依据；如后续评审发现新的边界场景，请在此文档追加记录。
